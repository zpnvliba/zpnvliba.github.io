ReentrantLock原理

ReentrantLock结构

  • ReentrantLock实现了Lock接口
  • 内部类FairSync和NonfairSync继承自Sync，重写了tryAcquire函数
  • Sync继承自AbstractQueuedSynchronizer,重写了tryRelease函数

非公平锁实现原理

默认是非公平锁

流程图:

**Lock方法获取锁**

流程：

  1. lock获取锁 :smiley:
  2. 获取成功，调用setExclusiveOwnerThread()函数将当前线程设置成锁持有者，结束:kissing_heart: 获取失败进入步骤3:relieved:
  3. 调用acquire()函数，如果这个时候恰好可以获取到，将当前线程设置成锁持有者，结束:kissing_heart: 否者进入步骤4:relieved:
  4. 调用子类tryAcquire()函数，然后接着调用nonfairTryAcquire()函数 非公平获取锁函数:kissing_heart:
  5. 获取当前线程名、获取当前state值，如果这个时候state恰好为0，那就获取锁，如果获取成功，将当前线程设置成锁持有者，结束:kissing_heart:否则进入步骤6:relieved:
  6. 如果当前线程已经持有锁，将state加一，结束:kissing_heart:
  7. 否则返回false，老老实实去同步队列排队吧:grimacing:

tryRelease锁的释放

流程图：


代码

*流程

  1. 判断当前线程是不是持有锁线程，如果不是直接抛异常。否则进入步骤2
  2. 将当前state减1，判断是不是等于0，如果等于0则将锁的持有者设置为null然后返回true,然后进入步骤3，不等于0进入步骤4
  3. 唤醒同步队列
  4. 不唤醒同步队列

公平锁实现原理

流程图

代码:

步骤；

  1. 尝试获取锁，state是否等于0，是进入步骤2，不是进入步骤3
  2. 判断同步队列是否有进程，有，返回false结束 return false，没有进入步骤4
  3. 判断当前锁持有者是否为当前线程，是，进入步骤5 不是 结束 return false
  4. 将当前线程设置成锁持有者，state = 1 return true
  5. 将state加一 return true

TryLock实现原理 （设置了超时时间的时候）

流程图

代码：

原理

  1. 判断当前节点的前驱节点是否为头结点，如果是，然后获取锁，如果获取成功，返回true, 如果没有获取到锁，到步骤2。如果不是，到步骤3
  2. 根据计时器判断是否超时，如果超时返回false ,否则加入到等待队列中
  3. 判断当前节点是否超时，超时返回false 结束，否则执行shouldParkAfterFailedAcquire()函数，告诉当前节点的前驱节点，我要进入等待状态了，到了记的叫我:joy:
  4. 执行LockSupport.parkNanos方法，将当前线程设置成超时等待的状态
